========================================
  MonsterOrderWilds 部署指南
========================================

问题：Windows Defender SmartScreen 阻止程序运行
解决方案：使用安装程序部署，可选择安装签名证书


========================================
  方式1：使用安装程序（推荐）
========================================

  构建安装程序（开发者）
----------------------------------------

前提条件：
- 已安装 Inno Setup 6（下载：https://jrsoftware.org/isdl.php）
- 已完成 Release 编译

步骤：
1. 进入 installer 目录
2. 双击运行 build-installer.bat
3. 等待构建完成
4. 安装包输出到 installer/output/ 目录


  运行安装程序（用户）
----------------------------------------

1. 双击 MonsterOrderWilds-Setup-*.exe
2. 选择安装目录
3. 选择是否安装证书（推荐安装）
4. 完成安装


========================================
  方式2：手动部署
========================================

  步骤1：复制文件到目标机器
----------------------------------------

需要复制以下文件：
1. MonsterOrderWilds.exe          (程序主文件)
2. MonsterOrderWildsGUI.dll       (GUI库)
3. MonsterOrderWilds.cer          (证书文件，可选)
4. 安装证书.bat                   (自动安装脚本，可选)


  步骤2：安装证书（重要！）
----------------------------------------

在目标机器上安装证书，有两种方式：


  方式A：使用自动安装工具（推荐）
----------------------------------------

1. 确保以下文件在同一目录：
   - MonsterOrderWilds.cer     (证书文件)
   - 安装证书.bat              (安装脚本)

2. 右键点击"安装证书.bat"

3. 选择"以管理员身份运行"

4. 等待安装完成，看到"Certificate installed successfully!"

5. 关闭窗口


  方式B：手动安装证书
----------------------------------------

1. 双击 MonsterOrderWilds.cer 文件
2. 点击"安装证书"按钮
3. 选择存储位置：
   - 选择"本地计算机"
   - 点击"下一步"
4. 选择证书存储：
   - 选择"将所有的证书都放入下列存储"
   - 点击"浏览"
   - 选择"受信任的根证书颁发机构"
   - 点击"确定"
5. 完成安装：
   - 点击"下一步"
   - 点击"完成"
6. 如果弹出安全警告，点击"是"


  步骤3：运行程序
----------------------------------------

完成证书安装后，双击运行 MonsterOrderWilds.exe

SmartScreen 警告应该：
- 完全消失，或者
- 显示"已验证的发布者"，可以安全点击"运行"


========================================
  备选方案：临时绕过
========================================

如果不想安装证书，可以临时绕过：

1. 双击 MonsterOrderWilds.exe
2. SmartScreen 弹出警告
3. 点击"更多信息"
4. 点击"仍要运行"

注意：每次程序更新后都需要重复此操作


========================================
  技术说明
========================================

• 程序已使用自签名证书签名
• 证书有效期：5年
• 证书信息：CN=MonsterOrderWilds
• 签名时间戳：DigiCert TSA

自签名证书 vs 商业证书：
┌─────────────────┬─────────────────┬─────────────────┐
│ 特性            │ 自签名证书      │ 商业证书        │
├─────────────────┼─────────────────┼─────────────────┤
│ 成本            │ 免费            │ ¥500-5000/年    │
│ SmartScreen信任 │ 需要安装证书    │ 立即信任        │
│ 适用场景        │ 内部/小范围     │ 公开发布        │
│ 证书管理        │ 手动分发        │ 自动信任        │
└─────────────────┴─────────────────┴─────────────────┘

安装程序特性：
• 不修改注册表
• 不创建快捷方式
• 不注册卸载程序
• 升级时保留 MonsterOrderWilds_configs 配置目录


========================================
  故障排除
========================================

问题1：安装证书后仍被阻止
解决：
1. 确认证书安装到"受信任的根证书颁发机构"
2. 重启计算机
3. 检查程序签名：
   - 右键点击 MonsterOrderWilds.exe
   - 选择"属性"
   - 查看"数字签名"选项卡

问题2：提示"证书链不完整"
解决：
1. 重新安装证书
2. 确保选择"受信任的根证书颁发机构"

问题3：程序无法启动
解决：
1. 确保 MonsterOrderWildsGUI.dll 与 exe 在同一目录
2. 检查 .NET Framework 4.6.1 是否已安装

问题4：Inno Setup 构建失败
解决：
1. 确认已安装 Inno Setup 6
2. 确认 Release 编译已完成
3. 检查 x64\Release 目录是否存在 exe 和 dll
