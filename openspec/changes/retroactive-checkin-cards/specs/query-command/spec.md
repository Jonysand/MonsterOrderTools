## ADDED Requirements

### Requirement: 补签查询命令处理
用户发送"补签查询"弹幕时，系统回复当前持有的补签卡数量和下次领取条件。

#### Scenario: 用户查询补签卡状态（持有卡片且可继续领取）
- **GIVEN** 用户 "user1" 当前持有2张补签卡
- **AND** 已连续点赞3天（需再点赞4天获得规则1奖励）
- **AND** 今日已点赞800（还需200达到1000）
- **AND** 本月尚未领取首次1000奖励
- **WHEN** 用户发送弹幕 "补签查询"
- **THEN** 回复/播报："user1，你当前持有2张补签卡。连续点赞7天奖励：已连续点赞3天，再点赞4天可获得1张。本月首次点赞1000奖励：今日已点赞800/1000，本月尚未领取。"

#### Scenario: 用户查询补签卡状态（持有卡片且本月已领取）
- **GIVEN** 用户 "user1" 当前持有1张补签卡
- **AND** 已连续点赞5天
- **AND** 本月已领取首次1000奖励（monthly_first_claimed=20260424）
- **WHEN** 用户发送弹幕 "补签查询"
- **THEN** 回复/播报："user1，你当前持有1张补签卡。连续点赞7天奖励：已连续点赞5天，再点赞2天可获得1张。本月首次点赞1000奖励：本月已领取。"

#### Scenario: 用户查询补签卡状态（无卡片且刚跨月）
- **GIVEN** 用户 "user1" 当前持有0张补签卡
- **AND** 连续点赞天数为0（今天第一次点赞）
- **AND** 今日已点赞50
- **AND** 新月份刚开始
- **WHEN** 用户发送弹幕 "补签查询"
- **THEN** 回复/播报："user1，你当前持有0张补签卡。连续点赞7天奖励：已连续点赞0天，再点赞7天可获得1张。本月首次点赞1000奖励：今日已点赞50/1000，本月尚未领取。"

#### Scenario: 用户查询补签卡状态（已达到所有条件）
- **GIVEN** 用户 "user1" 当前持有3张补签卡
- **AND** 已连续点赞6天（明天即可领取规则1）
- **AND** 今日已点赞999（再点赞1个即可领取规则2）
- **AND** 本月尚未领取首次1000奖励
- **WHEN** 用户发送弹幕 "补签查询"
- **THEN** 回复/播报："user1，你当前持有3张补签卡。连续点赞7天奖励：已连续点赞6天，再点赞1天可获得1张。本月首次点赞1000奖励：今日已点赞999/1000，再点赞1个即可领取！"

#### Scenario: 用户使用非标准查询词
- **WHEN** 用户发送弹幕 "补签卡查询"、"查询补签卡"、"我的补签卡" 等变体
- **THEN** 使用模糊匹配或同义词列表识别为查询命令
- **AND** 返回相同的查询结果
