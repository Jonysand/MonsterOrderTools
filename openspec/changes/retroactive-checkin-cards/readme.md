# 补签卡机制 - 新增弹幕指令说明

## 新增弹幕指令

### 1. 补签

**触发词：** `补签`、`补签卡`

**作用：**
- 用户发送此弹幕时，系统验证用户是否持有补签卡
- 若持有补签卡，扣除 1 张，并在 `checkin_records` 表中插入最近一个断签日期的打卡记录
- 重新计算用户的连续打卡天数
- 通过气泡消息和语音播报回复操作结果

**回复示例：**
- 成功：`user1 已成功补签4月23日，剩余补签卡1张，连续打卡恢复为5天！`
- 无卡：`user1，你没有补签卡哦~`
- 无断签：`user1，当前没有需要补签的日期。`

---

### 2. 补签查询

**触发词：** `补签查询`、`补签卡查询`、`查询补签`、`查询补签卡`、`我的补签卡`

**作用：**
- 用户发送此弹幕时，系统查询并回复当前持有的补签卡数量
- 同时显示领取进度：
  - **连续点赞7天奖励**：当前连续点赞天数 / 7天，差几天可领取
  - **本月首次点赞1000奖励**：今日已点赞数 / 1000，差几个可领取（每月限领1次，跨月重置）

**回复示例：**
```
user1，你当前持有2张补签卡。连续点赞7天奖励：已连续点赞3天，再点赞4天可获得1张。本月首次点赞1000奖励：今日已点赞800/1000，本月尚未领取。
```

---

## 补签卡获取规则

用户无需发送指令，系统会自动根据点赞行为发放补签卡：

### 规则1：连续7天点赞
- 用户连续7天每天点赞（like_count > 0），第7天结束时自动获得1张补签卡
- 连续14天、21天...每满7天周期均可获得1张

### 规则2：每月首次点赞1000
- 每月任意一天首次累计点赞达1000，立即获得1张补签卡
- 每月仅可领取1次，跨月自动重置

**发放通知：** 系统自动通过气泡和语音播报通知用户获得补签卡。

---

## 注意事项

- 所有指令使用**精确全等匹配**（非子串匹配），弹幕内容必须完全等于触发词才能触发
- 每次"补签"只能补最近的一个断签日期，如有多个断签需多次发送
- 补签卡永久有效，无过期时间
- 点赞数据按日期独立累计，昨天的点赞数不计入今天
