## idCode 注册表持久化规格说明

### 概述

idCode（身份码）用于 B 站弹幕连接的用户身份验证。采用 Windows 注册表进行持久化存储，提高数据可靠性。

### 注册表配置

| 项目 | 值 |
|------|-----|
| 根键 | HKEY_CURRENT_USER |
| 子键 | Software\MonsterOrderWilds |
| 值名 | IdCode |
| 类型 | REG_SZ (字符串) |

### 数据流

```
【读取流程】
程序启动 → LoadConfig()
  → 从 JSON 加载其他配置项
  → 从注册表读取 idCode
  → 完成配置初始化

【写入流程】
用户修改 idCode → SaveConfig()
  → 其他配置写入 JSON 文件
  → idCode 写入注册表
  → 完成配置保存
```

### 兼容性

- **向后兼容**：JSON 配置文件中的 `ID_CODE` 字段保留，但不再用于读取
- **首次运行**：如果注册表中不存在 `IdCode` 值，返回空字符串
- **数据迁移**：旧版本用户的 idCode 仍然可以从 JSON 配置文件中手动恢复

### 错误处理

| 场景 | 处理方式 |
|------|----------|
| 注册表键不存在 | 返回空字符串，不影响程序启动 |
| 注册表访问被拒绝 | 记录错误日志，idCode 使用内存值 |
| 注册表写入失败 | 记录错误日志，保存操作失败 |

### 安全考虑

- idCode 仅存储在当前用户注册表，不影响其他用户
- 不需要管理员权限
- 注册表数据受 Windows 用户账户控制 (UAC) 保护
